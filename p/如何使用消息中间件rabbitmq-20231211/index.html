1 简述一下消息队列

消息队列（Message Queue）是一个程序之间的通信方法。在开发过程中，我们一般将耗时且无需即时返回的操作进行异步处理，从而提高系统的吞吐量，提高程序之间的耦合度。消息队列有两种实现方式分别：AMQP、JMS。

常见的消息队列服务：

  4. RocketMQ：阿里巴巴提供的。
  5. Kafka：一般用在大数据场合。

title: JMS和AMQP的区别
- JMS是定义了统一的接口，而AMQP是定义了统一的数据传输协议。
- JMS限定了必须使用Java语言，而AMQP只是协议，所以跨语言。
- JMS广泛上定义了两种消息模式，而AMQP的消息模式更丰富。

package com.cwg.rabbitmq.simple;  
  
import com.rabbitmq.client.Channel;  
import com.rabbitmq.client.Connection;  
import com.rabbitmq.client.ConnectionFactory;  
  
import java.io.IOException;  
import java.util.concurrent.TimeoutException;  
  
/**  
 * 简单模式：发送消息  
 */  
public class Producer {  
    static final String QUEUE_NAME = "simple_queue";  
  
    public static void main(String[] args) throws IOException, TimeoutException {  
        //1.创建连接工厂
        ConnectionFactory connectionFactory = new ConnectionFactory();  
        // 主机  
        connectionFactory.setHost("192.168.1.201");  
        // 端口  
        connectionFactory.setPort(5672);  
        // 用户名  
        connectionFactory.setUsername("admin");  
        // 密码  
        connectionFactory.setPassword("admin");  
        //2.创建连接  
        Connection connection = connectionFactory.newConnection();  
        //3.创建频道  
        Channel channel = connection.createChannel();  
        //4.声明队列  
        /**  
         * 参数1：队列名称  
         * 参数2：是否定义持久化队列  
         * 参数3：是否独占本连接  
         * 参数4：是否再不使用的时候队列自动删除  
         * 参数5：其他参数  
         */  
        channel.queueDeclare(QUEUE_NAME, true, false, false, null);  
        //5.发送消息到队列  
        /**  
         * 参数1：交换机名称，如果没有则指定空字符串(表示使用默认的交换机)  
         * 参数2：路由key，简单模式中可以使用队列名称  
         * 参数3：消息其他属性  
         * 参数4：消息内容 
         */  
        channel.basicPublish("", QUEUE_NAME, null, "你好！消费者".getBytes());  
        System.out.println("已发送消息：你好！消费者");  
        //6.关闭资源  
        channel.close();  
        connection.close();  
    }  
}

public class Consumer {  
    public static void main(String[] args) throws IOException, TimeoutException {  
        //1. 创建连接  
        Connection connection = ConnectionUtil.getConnection();  
        //2. 创建频道  
        Channel channel = connection.createChannel();  
        //3. 声明队列  
        channel.queueDeclare(Producer.QUEUE_NAME,true,false,false,null);  
        //4. 创建消费者  
        DefaultConsumer defaultConsumer = new DefaultConsumer(channel){  
	        //4.1 处理接收到的方法
            @Override  
            public void handleDelivery(String consumerTag, Envelope envelope, AMQP.BasicProperties properties, byte[] body) throws IOException {  
                //输出路由信息  
                System.out.println("路由key："+envelope.getRoutingKey());  
                //输出交换机信息  
                System.out.println("交换机信息："+envelope.getExchange());  
                //消息id  
                System.out.println("消息id："+envelope.getDeliveryTag());  
                //接收到的消息  
                System.out.println("接收到的消息："+ new String(body,"utf-8"));  
            }  
        };  
        //5. 监听队列  
        /**  
         * 参数1：队列名  
         * 参数2：是否要自动确认，设置为true表示消息接收到自动向MQ回复接收到了，MQ则会将消息从队列中删除，如果为false就需要手动确认。  
         * 参数3：哪个消费者  
         */  
        channel.basicConsume(Producer.QUEUE_NAME,true,defaultConsumer);  
    }  
}

3.4 小结

简单模式：生产者发送消息到队列(simple_queue)中，一个消费者从队列中接收消息。

再RabbitMQ中消费者只能从队列接收消息。 如果接收消息的消费者在同一个队列中有两个或者多个时，消息又是如何分配的？

4 RabbitMQ工作模式

4.1 工作模式

Work Queues与简单模式相比，多了一个消费者或多个消费者，而多个消费者之间对于接收消息是竞争关系。 一个消息只能被一个消费者接收，其他消费者不能接收同一条消息。该模式可以应用在，消费端处理任务比较耗时的时候，添加对同一个队列的消费者来提高任务处理能力。

4.2 订阅模式

在订阅模式中存在四种角色：

  1. 生产者(Producer)：也就是要进行发送的程序，但是不在发送给消息队列，而是发给交换机。
  2. 消费者(Consume)：消息的接收者，会一直等待消息的到来。
  3. 消息队列(Queue)：接收消息，缓存消息。
  4. 交换机(Exchange)：交换机。一方面接收生产者发送的消息。另一方面，知道如何处理消息，比如丢弃，单独传递，传递所有等操作，这些操作取决于交换机的类型。交换机有常见的3种类型
    1. 广播(Fancut)：将消息发给所有绑定到交换机的队列。
    2. 定向(Direct)：将消息发给符合指定routing key的队列。
    3. 通配符(Topic)：将消息发给符合routing pattern(路由模式) 的队列。

注意：交换机只支持转发消息，不支持缓存消息。

4.3 Routing路由模式

用来更加精确的传输消息，将队列与routingkey进行绑定，在发送消息时，声明路由key，从而能让对应的队列进行接收到消息

4.4 Topic通配符模式

Topic类型和Direct相比，都是可以根据RoutigKey把消息路由分配到不同的队列。只不过Topic Exchange(通配符交换机)可以让队列在绑定Routing key的时候使用通配符。
