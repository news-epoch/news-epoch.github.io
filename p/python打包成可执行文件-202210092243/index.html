title: 打包成单文件
- 打单文件包 `pyinstaller -F 程序启动文件 -n 可执行文件别名`
	- -w: 隐藏控制台
	- -n：别名
	- -F：将多个文件打包到一个指定的文件中
	- -i: 使用图标（后面接该文件夹下的图标文件名）
````ad-warning
title: 打包单文件的路径问题
```Python
# 一般来说我们对代码处理文件路径会对其进行拼接，从而得到文件路径
import os
BASE_DIR = os.path.dirname(os.path.abspath(__file__))
filePath = os.path.join(BASE_DIR,'utils\chromedriver.exe')
# 但是在打包成单路径后，运行执行文件，会进行解压并且放在临时目录下，从而导致路径发生改变
## pyinstaller提供了两种查询方法
import sys
### 获取命令运行py文件脚本目录  python py文件目录，由此可得到可执行文件的目录
sys.argv[0]
### 获取这个文件的绝对路径
BASE_DIR = os.path.realpath(sys.argv[0])
### 获取这个文件的目录的绝对路径
BASE_DIR = os.path.dirname(os.path.realpath(sys.argv[0]))
### 拼接路径
filePath = os.path.join(BASE_DIR,'utils\chromedriver.exe')
```
````

防止文件被反编译出来

  1. 防止python制作的exe被反编译出来
  2. 安装pip install Cython
  3. 报错Unable to find vcvarsall.bat安装vc(Pyhon不同的版本需要安装相应的vs才能顺利将py转成pyd)先安装 vs_professional__2017.exe，再安装 visualcppbuildtools_full.exe
  4. 创建一个setup.py 
    1
2
3
4
5

    from distutils.core import setup
from Cython.Build import cythonize
setup(
ext_modules = cythonize(["add.py"]), # add.py 为需要打包的文件名，不能包含中文
)
  5. 窗口执行python setup.py build_ext --inplace
  6. 按回车后，会生成 add.cp38-win_amd64.pyd，修改文件名为add.pyd，即删除多的【.cp38-win_amd64】
  7. 在启动文件导入add.pyd包
  8. installer -F 执行文件
