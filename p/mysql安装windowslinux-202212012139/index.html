title: 忘记root密码时
1. 第一种
	- 修改my.cnf文件,**\[mysqld\]下**添加一行，使其登录时跳过权限检查`skip_grant_table`
	- 重启数据库
	- 启动MySQL服务，登录MySQL，此时提示输入密码，输入任意密码回车即可进入MySQL。
2. 第二种
	- 关闭数据库
	- `mysqld --skip-grant-tables`
	- 登录MySQL，此时提示输入密码，输入任意密码回车即可进入MySQL。

title: ERROR 1819 (HY000): Your password does not satisfy the current policy requirements

这个其实与validate_password_policy的值有关。
alidate_password_policy有以下取值：默认是1，即MEDIUM，所以刚开始设置的密码必须符合长度，且必须含有数字，小写或大写字母，特殊字符。
有时候，只是为了自己测试，不想密码设置得那么复杂，譬如说，我只想设置root的密码为123456。
首先，修改validate_password_policy参数的值

mysql> set global validate_password_policy=0;

判断密码的标准就基于密码的长度了。这个由validate_password_length参数来决定。

mysql> select @@validate_password_length;
mysql> SET GLOBAL validate_password_length=6;

Docker安装mysql8.0.23无法远程登录

  1. 查询插件select host,user,plugin from user; ![[1690463430958.png]]
  2. 更改密码插件update user set plugin='mysql_native_password' where user='root'; ![[1690463492719.png]]
  3. 查看是否有远程登录，没有就创建一个远程登录账户create user 'root'@'%' identified with mysql_native_password BY 'Mysql123';
  4. 修改密码ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY 'Mysql123';
  5. 刷新权限flush privileges; ，注意上面步骤执行完，才能刷新权限，不然会造成无法登录的情况。
