编程式事务控制的三大对象

1 PlatformTransactionManager

PlatformTransactionManager接口时spring的事务管理器，它里面提供了我们常用的操作事务的方法。

方法 说明
TransactionStatus getTransaction(TransactionDefination defination) 获取事务的状态
void commit(TransactionStatus status) 提交事务
void rollback(TransactionStatus status) 回滚事务
注意：PlatformTransactionManager 是接口类型 ，不同的Dao层技术则有不同的实现类，例如Dao层技术是jdbc或mybatis时，org.springframework.jdbc.datasource.DatasourceTransactionManager
Dao层技术是hibernate时，org.springframework.orm.hibernate5.HibernateTransactionManager

2 事务在Spring中的传播行为TransactionDefinition

方法 说明
int getIsolationLevel 获取事务的隔离级别
int getPropogationBehavior 获取事务的传播行为
int getTimeout 获取超时时间
boolean isReadOnly 是否只读

开启事务A时，在这个事务A中需要在开一个事务B，这时就需要选择，是在开一个事务B，还是就在当前事务进行执行。这就是事务的传播行为

  • 而事务的传播行为一共有7种，其中最常用的有3种，并且默认是1秒的超时时间：
    • REQUIRED(默认)：如果没有事务，就开启一个事务，如果已经存在一个事务了，那么就加入到当前事务中
    • REQUIRED_NEW：如果已经存在一个事务，那么就挂起当前事务，重新在开一个事务
    • NOT_SUPPORTED：如果当前存在事务，就把当前事务挂起。

        </aop:advisor>

package com.cwg.dao.impl;  
  
import com.cwg.dao.AccountDao;  
import org.springframework.beans.factory.annotation.Autowired;  
import org.springframework.jdbc.core.JdbcTemplate;  
import org.springframework.stereotype.Repository;  
  
import javax.annotation.Resource;  
  
@Repository("accountDao")  
public class AccountDaoImpl implements AccountDao {  
  
    @Resource(name = "jdbcTemplate")  
    private JdbcTemplate jdbcTemplate;  
  
    public JdbcTemplate getJdbcTemplate() {  
        return jdbcTemplate;  
    }  
  
    public void setJdbcTemplate(JdbcTemplate jdbcTemplate) {  
        this.jdbcTemplate = jdbcTemplate;  
    }  
  
    @Override  
    public void in(String inMan, double money) {  
        jdbcTemplate.update("update account set money=money+? where account.name=?",money,inMan);  
    }  
  
    @Override  
    public void out(String outMan, double money) {  
        jdbcTemplate.update("update account set money=money-? where account.name=?", money, outMan);  
    }  
}
