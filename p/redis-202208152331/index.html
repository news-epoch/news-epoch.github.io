Redis的安装

二进制安装

  1. 下载redis安装包
  2. 解压安装包，并复制到/usr/local/下
  3. yum -y install gcc gcc-c++ make
  4. cd /usr/local/redis/src/
  5. make && make install
  6. cd /usr/local/redis/ && vim redis.conf
  7. redis-server redis.conf

Redis事务

概念

为了防止在多用户操作写数据出现错误，为执行实现的功能。

悲观锁

不相信别人不会在你访问的时间内，同时对表进行访问，就对数据进行了锁表操作，在用户读取该表时，不允许任何用户对这个表进行读写

乐观锁

相信其他用户不会在你访问你的时间内，对表进行读写，就在每次修改表后，对表进行更新版本操作，这样其他人在后1s进行写操作就会报错，需要重写读取。类似于网络中的总线型。

Redis持久化

概念

redis如何进行的备份数据，在主线程的基础上加一条fork子进程，通过子进程先将数据写入一个临时文件，等持久化结束后，修改临时文件名替换原备份文件，在这个期间主进程是不进行任何读写操作的，确保了极高的性能。

RDB

在指定时间内向RDB文件写入数据集快照，恢复时直接将快照文件直接读入内存中，是Redis数据库more默认的存储方式，当出现服务意外宕机，会丢失最后一次存储的数据集。

title: 具体实现

1. `save 600 1` 在600s内往里面修改了1条数据就进行持久化
2. `bgsave` 直接手动执行持久化
3. `rdbcompression yes` 以rdb进行存储时，对其进行压缩

AOF

以日志的形式记录下每一个操作，只能进行追加(不会对读操作进行写入)，完整性比RDB文件高，但是比较消耗性能和内存，但是安全性不高。当AOF文件占用过高可以对AOF文件进行重写，

1. `appendonly yes` 开启AOF持久化
