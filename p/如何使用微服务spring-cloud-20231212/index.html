1 简述微服务架构

在SOA架构中使用了ESB组件的面向服务架构，由于ESB自身服务复杂；应用服务粒度较大，所有服务之间的通信都经过ESB，会降低通信的速度，部署、测试ESB又比较麻烦。所以由此衍生出微服务架构。 什么是微服务架构呢？微服务架构是使用一套小服务来开发单个应用的方式或途径。主要特点有：单一职责，服务粒度小，面向服务(对外暴露REST api)，服务之间相互独立。通过HTTP协议进行调用。

5 Eureka注册中心

Eureka注册中心负责管理、记录服务，监控提供者的信息(提供者自身的元数据，如IP地址、端口、运行状况指示符URL等信息)，服务调用者无需自己寻找服务，而是把自己的需求告诉Eureka，然后Eureka会把符合你需求的服务告诉你。Eureka就好比如是滴滴，而载客的私家车则是服务提供者，我们将私家车都到滴滴注册，记录车的信息，此时，打车的人，只需要打开APP，输入目的地，选择车型(服务类型)，滴滴就会自动安排。

@RestController  
@RequestMapping("/consummer")  
public class UserController {  
    @Autowired  
    private RestTemplate restTemplate;  
  
    @Autowired  
    private DiscoveryClient discoveryClient;  
    @GetMapping("/{id}")  
    public User queryById(@PathVariable Long id){  
        List<ServiceInstance> serviceInstances = discoveryClient.getInstances("user-service");  
        ServiceInstance serviceInstance = serviceInstances.get(0);  
        String URL = "http://" + serviceInstance.getHost() +":"+serviceInstance.getPort()+"/user/"+ id;  
        System.out.println("URL = " + URL);  
        return restTemplate.getForObject(URL,User.class);  
    }  
}

5.4 小结

类/注解/配置 作用
@RestController 标识用于返回数据的Controller
@EnableDiscoveryClient 开启Eureka客户端发现功能
@EnableEurekaClient 标识该服务是要连接Eureka的客户端
@EnableEurekaServer 标识该服务是Eureka服务
new DiscoveryClient() 用来获取注册服务的各种信息(Host,Port等)
eureka.client.service-url.defaultZone eureka 服务地址，如果是集群的话，需要指定其他集群eureka地址
eureka.client.register-with-eureka 是否注册自己到Eureka服务上(集群需要注册自己)
eureka.client.fetch-registry 要不要获取具体的服务,应该不拉取服务。
eureka.client.instance.prefer-ip-address 默认注册地址：更倾向于使用ip地址，而不是host名
eureka.client.instance.ip-address 注册的IP地址为
eureka.client.instance.lease-renewal-interval-in-seconds 心跳持续时间，默认30s，(每隔30s，定时向eureka服务端发送请求，告知自己还活着)
eureka.client.instance.lease-expiration-duration-in-seconds 默认90s内没有给服务器发送心跳，就剔除该服务。

7 Hystrix熔断器

由于在微服务中，服务间调用关系错综复杂，一个请求可能需要调用多个微服务接口才能实现，会形成非常复杂的调用链路，如果在调用过程中，有一个服务出现异常，请求阻塞，用户请求就不会得到响应，则tomcat这个线程就不会释放，于是越来越多的用户请求到来，越来越多的线程会被阻塞，从而造成雪崩效应。由此形成一种为Hystrix熔断器机制，Hystrix解决雪崩问题的主要手段是：服务降级(及时返回服务调用失败的结果，让线程不因为等待服务而阻塞)、线程隔离(用户请求不直接访问服务，而是使用线程池空闲的线程，访问服务)，服务熔断。

7.1 线程隔离与服务降级

7.1.1 线程隔离原理

用户请求不直接访问服务，而是使用线程池空闲的线程，访问服务。

  • Hystrix为每个依赖服务分配一个小的线程池，如果线程池已满，就立刻拒绝，默认不采用排队，从而加速失败的判定时间。
  • 用户的请求将不再直接访问服务，而是通过线程池中的空闲线程来访问服务，如果线程池已满，或者请求超时，则会进行降级处理(服务降级：优先保证核心服务，而非核心服务不可用或弱可用)。
  • 服务降级虽然会导致请求失败，但是不会导致阻塞，而且最多会影响这个依赖服务对应的线程池中的资源，对其他服务没有响应。

public class MyParamGatewayFilterFactory extends AbstractGatewayFilterFactory<MyParamGatewayFilterFactory.Config> {  
    static final String PARAM_NAME = "name";  
  
    public MyParamGatewayFilterFactory() {  
        super(MyParamGatewayFilterFactory.Config.class);  
    }  
  
    public List<String> shortcutFieldOrder() {

    }  
  
    @Override  
    public GatewayFilter apply(Config config) { //#TODO  
        return ((exchange, chain) -> {  
            // 获取请求参数中的param中对应的参数名的参数值。  
            ServerHttpRequest request = exchange.getRequest();  
            if (request.getQueryParams().containsKey(config.name)) {  
                request.getQueryParams().get(config.name).forEach(value -> System.out.printf("----------------局部过滤器--------%s = %s--------", config.name, value));

            return chain.filter(exchange);  
        });  
    }

import org.springframework.core.Ordered;  
import org.springframework.http.HttpStatus;  
import org.springframework.stereotype.Component;  
import org.springframework.web.server.ServerWebExchange;  
import reactor.core.publisher.Mono;  
  
/**  
 * @program: HomeJava  
 * @description:  
 * @authro: Mr.chen  
 * @create: 2023-12-16 20:58  
 **/@Component  
public class MyGlobalFilter implements GlobalFilter, Ordered {  
    @Override  
    public Mono<Void> filter(ServerWebExchange exchange, GatewayFilterChain chain) {  
        System.out.println("---------这是一个全局过滤器------------------");  
        String token = exchange.getRequest().getQueryParams().getFirst("token");  
        System.out.println("---------------------------"+token+"_____________________________________________");  
        if (StringUtils.isBlank(token)){  
            // 设置响应状态码为未授权  
            exchange.getResponse().setStatusCode(HttpStatus.UNAUTHORIZED);  
            // 完成  
            return exchange.getResponse().setComplete();  
        }  
        return chain.filter(exchange);  
    }  
  
    @Override  
    public int getOrder() {  
        // 值越小越先执行  
        return 1;  
    }  
}

Spring Cloud Bus 是用轻量的消息代理将分布式的节点连接起来，可以用于广播配置文件或者服务的监控管理。也就是消息总线可以为微服务做监控，也可以实现应用程序之间的相互通信。Spring Cloud Bus 可选的消息代理有RabbitMQ和Kafka。应用场景1：在Spring Cloud Config中修改了配置文件，在不重启为服务的情况下，让配置文件生效。
